## V2-M3 飞升点：化神圆满后的长期挂机 sink

对应 spec §6。前置：M2 已落 buff 合算 clamp（飞升被动必须进 clamp）。

### 骨架任务
- **T3.1 数据表**：`ascension(user_id PK, level, points, spent_json, updated_at)`（spec §9.2），幂等建表。
- **T3.2 飞升点获取**：化神圆满溢出修为转换（复用 M2 的 `overflow_split`：越界修为 ×30%→道行、×20%→飞升点，其余 sink 损耗）、每周飞升试炼、化神 Boss 前列、活动副本兑换。
- **T3.3 飞升试炼**：化神圆满解锁；消耗长期资源（道行/飞升材料）；非三期境界、不推进炼虚。
- **T3.4 账号级被动**：气血/闭关/炼制等 +1%/级，**单项硬上限 5 级**（spec §6.3）；接入 M2 clamp 层（飞升加成计入合算上限，不得旁路）。
- **T3.5 称号/播报**：`game_events` 发飞升成功，解锁称号/群播报特殊文案。

### 验收锚点
- `ascension` 表幂等；飞升点不可交易。
- 单项被动 **5 级硬上限**（超过不再生效）——专门单测。
- 飞升加成进入全局属性**仍受 §6.3 合算上限**（回归 M2 `test_buff_caps.py` + M1 化神门槛）。
- 化神圆满溢出修为按 `overflow_split` 分流：cultivation 封顶，越界 ×30%→道行、×20%→飞升点，其余损耗。

### 待澄清（启动前定稿）
- 已定稿：化神圆满溢出修为 `30%` 转道行、`20%` 转飞升点、`50%` sink 损耗；不再双倍发放。

## V2-M4 活动副本与宗门战 / 赛季

对应 spec §5.4、§8。前置：M2 绑定库存（活动材料绑定）。不阻塞，群活跃增强。

### 骨架任务
- **T4.1 活动副本**：每周轮换主题（天魔潮/丹霞会/万剑冢）；活动材料绑定不可交易；活动商店兑换称号/转修令/保命符/道途材料/外观；给道行但设**周上限**（防肝度失控）。
- **T4.2 宗门战**：每周固定窗口（初版周六 20:00~21:00）；据点 3 个（闭关/掉落/灵石加成）；复用战斗引擎打据点守卫，按宗门累计积分结算；**据点 buff 进 M2 合算上限**（与宗门福利叠加有总上限）。
- **T4.3 赛季天梯**：现周奖池升级为月赛季；赛季末发绑定称号/少量灵石/道行，**不发可交易关键材料**。

### 验收锚点
- 活动副本道行有**周上限**（超额不再发）——单测。
- 据点 buff + 宗门福利叠加**不破 §6.3 合算上限**（回归 clamp 测试）。
- 赛季奖励不含可交易关键材料（防经济注水）。
- 活动材料 `bound=1`，不可进交易/掠夺路径。

## V2-M5 玩家经济：一口价坊市（后置）

对应 spec §7。前置：M2 绑定库存已迁移。**最后做**——满级成长目标优先于交易。

### 骨架任务
- **T5.1 一口价坊市**先于拍卖行；**材料/丹药**先于法宝实例（spec §7.1）。
- **T5.2 交易表**（spec §9.2）：`market_listings(...)`；拍卖 `auctions`/`auction_bids` 留更后。
- **T5.3 交易税 + 审计**：上架/成交走 `db.transaction()` + `callback_tokens`；价格异常审计、撤单/结算测试。
- **T5.4 绑定隔离**：活动材料/飞升材料/转修令/保命符**默认绑定**，不可上架（复用 M2 `bound` 语义）。
- 稀有法宝进拍卖行前，必须先补交易审计 + 撤单/结算测试（spec §7.1）。

### 验收锚点
- 绑定物（`bound=1`）无法上架——单测。
- 成交灵石流向 + 交易税正确、原子；并发成交不超卖（复用 `db.transaction()` 串行化）。
- 玩家交易开放后，**反套利红线仍成立**（坊市不得绕过"买精力→刷钱"红线）——回归 `test_economy.py`。
- 撤单/结算/异常价审计有日志可查。

### 暂不进入 v2.0（spec §12）
- 炼虚/合体/大乘等更高境界、跨群世界 Boss/全服战、资源掠夺型生死斗、任何付费加速/突破/转修、完整道侣/师徒社交链。
- 生死斗（spec §8.3）延后到经济审计成熟；若做须双向确认+限额+冷却+不可夺绑定物+日志可查。

## 三段共同 DoD

1. 各自 `python -m pytest` 全绿，且**不回归 M0~M2 的化神门槛 / buff clamp / 反套利测试**。
2. 任何新增百分比加成（飞升被动、据点 buff）**一律走 M2 clamp 层**，禁止旁路全局属性计算。
3. 任何新增可交易/可兑换产出**一律进 balance_sim 反套利校验**。
4. 所有 schema 改动幂等、旧档无损（`_ensure_column` 或独立可重入迁移）。
