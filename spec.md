# 《问道》— 修仙主题 Telegram 群文字 RPG · 设计规格 (spec)

> 一款 Chat Wars 氛围、修仙世界规格的中文 Telegram 文字 RPG。
> 技术栈:Python + aiogram 3.x + SQLite(aiosqlite)。交互:指令 + inline 按钮。
> 首版(v1)= 个人养成核心循环 + PvE + PvP 群多人玩法。

文档状态:**设计定稿,待实施**。本文件是实现的唯一事实来源;数值标注"初版可调"者均需上线后实测调参。

## 2. 核心循环总览

```
        ┌──────────────────────────────────────────────────┐
        ↓                                                  │
  [闭关] 挂机·按真实时间攒修为(惰性结算·离线上限)            │
        ↓ 修为达标                                          │
  [突破境界] 小突破近自动 / 大突破需突破丹+成功率(金丹起渡天劫) │
        ↓ 解锁更高地图·功法·精力上限                          │
  [历练/秘境] 耗精力 · 即时自动战斗(PvE) · 掉落              │
        ↓ 材料/灵石/丹方/功法残页/法宝                        │
  [炼丹/炼器] 定时制造(可灵石加速) → 丹药 · 法宝             │
        ↓ 嗑药 + 装法宝 + 配功法                             │
  [变强] 更快闭关 + 更深秘境 ───────────────────────────────┘

  群分支:  切磋/天梯(纯竞技)   ·   宗门(福利/贡献/商店)   ·   世界Boss(合击)
```

养成主操作在 **DM**;群内承载切磋、世界Boss、排行、宗门播报。

## 3. 角色与属性

### 3.1 创建角色(/start · 测灵根)
- 绑定 `tg_user_id`,生成角色。
- **测灵根**:随机根骨数值(初版区间 `[40, 80]`,正态偏中),并赋予灵根品类文案(如"金灵根""五行杂灵根")。根骨可后续用洗髓丹/天材地宝提升。
- 初始:炼气一层、灵石 100、精力满、空储物袋、赠新手剑(凡阶)与基础战技 1 个。

### 3.2 属性(v1 共 7 项)

| 属性 | 作用 |
|---|---|
| 气血 (HP) | 生命,归零判负 |
| 法力 (MP) | 战技消耗资源,战斗内每回合微量回复 |
| 攻击 | 伤害主系数 |
| 防御 | 减伤(见 §7 公式) |
| 身法 | 出手顺序 + 闪避 |
| 暴击 | 暴击触发率与倍率 |
| **根骨**(元属性) | 养成资质,影响闭关收益与突破成功率,非战斗属性 |

**最终属性 = 境界基础 + 法宝加成 + 功法(心法)加成 + 丹药临时 buff + 宗门福利。**
所有属性服务器端计算,客户端不可信。

### 3.3 境界阶梯(v1)

| 大境界 | 小阶 | 解锁 |
|---|---|---|
| 炼气期 | 一~十三层 | 历练入门、后山地图 |
| 筑基期 | 初/中/后/圆满 | 炼丹炼器、宗门、妖兽森林 |
| 金丹期 | 初/中/后/圆满 | 万妖岭、秘境、**天劫初现** |
| 元婴期 | 初/中/后/圆满 | 上古战场、世界Boss高阶、神识流派 |
| 化神期+ | — | **二期** |

每个大境界基础属性显著跃升(初版示例,可调):

| 境界 | 气血基础 | 法力 | 攻击 | 防御 | 身法 | 暴击 |
|---|---|---|---|---|---|---|
| 炼气(满) | 500 | 100 | 60 | 40 | 30 | 10 |
| 筑基(满) | 1500 | 250 | 160 | 110 | 70 | 25 |
| 金丹(满) | 5000 | 600 | 450 | 320 | 160 | 60 |

小阶内按层数线性插值;具体数值进配置文件 `config/realms.py`。

## 4. 修炼系统

### 4.1 闭关(离线挂机修为)
- `/cultivate` → 进入闭关,记录 `seclusion_start_at`(服务器时间戳)。
- **惰性结算**:任何读取时计算
  `获得修为 = 当前小阶突破成本 × min(now − seclusion_start_at, 离线上限) / 24小时 × (1 + 根骨/200) × 地点/宗门/丹药系数`
  - 离线上限初版 **12 小时**(宗门福利/洞府可提升)。
  - 闭关基础收益按**当前小阶**计算;不计根骨/宗门/丹药等加成时,多次闭关累计约 **24 小时升 1 小阶**。
  - 拆分多次闭关时保留小数余量,避免短闭关反复损失收益。
  - 地点系数:普通洞府 1.0 / 聚灵阵 1.3 / 洞天福地(限时) 1.6(初版)。
- 闭关期间不可历练/秘境/挑战 Boss;群内切磋不受闭关影响;`/cultivate` 再次调用为"出关并领取"。
- 出关结算后,若修为达标提示可突破。

### 4.2 精力
- 上限:炼气 100 / 筑基 120 / 金丹 150 / 元婴 200(初版)。
- 恢复:**1 点 / 4 分钟**,惰性结算(记 `stamina`, `stamina_updated_at`)。
- 消耗:历练 10、高阶图 15、秘境 20/层、世界Boss 挑战 15。
- 补充:补灵丹、灵石购买(灵石 sink,后续可作付费点)。

### 4.3 突破
- **小突破**(层/阶内):修为达阈值即可,**成功率 100%**,消耗修为,瞬时完成。
- **大突破**(跨大境界):需 `修为满 + 突破丹`,**有成功率**:
  `成功率 = 基础 + 根骨修正 + 丹药/法宝/阵法加成`(上限 95%)
  - 基础初版:筑基 80% / 金丹 70% / 元婴 60%。
  - 根骨修正 = `(根骨 − 50) × 0.3%`。
- **失败(轻惩罚,不跌境)**:损失当前小阶修为的 **30%**,获得"道基不稳"debuff(属性 −10%,持续到服用疗伤丹或自然恢复 N 小时)。突破丹消耗。
- **天劫**(金丹、元婴大突破额外触发):进入一场特殊 PvE——承受 **3 道雷劫**(每道为一次对自身的高额伤害,可用护体法宝/护盾战技/疗伤丹扛过)。扛过 = 突破成功;倒下 = 视为突破失败(走上面轻惩罚)。保命符放二期。

## 5. PvE 内容

### 5.1 历练(日常刷)
- `/explore` → 选地图(按境界解锁)→ 消耗精力 → 遭遇 1~3 妖兽 → 即时自动战斗 → 掉落。
- 每张地图含:**妖兽池 + 掉落表 + 一只妖王(精英,低概率遭遇,掉好物)**。

| 地图 | 解锁境界 | 精力 | 妖王 |
|---|---|---|---|
| 青牛后山 | 炼气 | 10 | 千年青牛 |
| 妖兽森林 | 筑基 | 10 | 噬魂狼王 |
| 万妖岭 | 金丹 | 15 | 万妖之主·蛟 |
| 上古战场 | 元婴 | 15 | 残魂魔将 |

### 5.2 秘境(副本)
- `/dungeon` → 层层深入(如 5 层),每层耗精力,末层首领。
- **每日入场限次**(初版 1 次/秘境/日)。
- 掉落优于历练:丹方/图纸、功法残页、稀有法宝、天材地宝。

### 5.3 世界Boss(群合击 · lite)
- **群绑定实例**:每个启用机器人的群拥有独立世界Boss(群即副本,最简实现)。
- 定时刷新(初版每日 20:00,APScheduler),持续 2 小时或血量清空。
- 玩家在群内 `/boss` 挑战:按 Boss 档位消耗精力,造成伤害 = 自身一次战斗模拟的总输出;Boss 基础血量按境界档位配置,生成实例时再按本群已知修仙者人数缩放。
- 群内用**单条可编辑播报消息**滚动显示 Boss 血条与伤害榜,避免刷屏。
- 结算奖励:按**累计伤害排名**发放(灵石/材料/稀有丹方),另有**参与奖**。

### 5.4 掉落模型
掉落表按 `(物品, 权重, 数量区间)` 配置,稀有度分层:
材料(高频)· 灵石(高频)· 丹方/图纸残页(中)· 功法残页(中低)· 法宝(低)· 突破丹/天材地宝(极低)。

## 6. 物品与储物袋

### 6.1 五大类

| 类别 | 内容 | 机制 |
|---|---|---|
| 材料 | 草药/矿石/兽材/妖丹 | 高堆叠,制造原料,可 NPC 回收 |
| 丹药 | 突破丹·疗伤丹·补灵丹·增益丹·洗髓丹 | 消耗品 |
| 法宝 | 武器/护体/饰品 | **可装备实例**,带品阶 + 随机词条 |
| 功法 | 心法(被动) + 战技(主动) | 残页集齐解锁;装入功法栏 |
| 杂项 | 丹方/图纸/令牌/凭证 | 解锁配方与功能 |

### 6.2 装备与功法栏
- **装备槽**:主器(武器) ×1 · 护体 ×1 · 饰品 ×2。
- **功法栏**:心法 ×1(被动主属性) + 战技 ×3(进战斗)。
- **法宝品阶**:凡 / 灵 / 宝 / 仙(基础值递增 + 词条数 1~3)。
- **法宝词条**(随机):攻击%、气血%、防御%、吸血%、反伤%、先攻、暴击+、抗暴、破甲。
- **功法品级**:黄 / 玄 / 地 / 天阶。

### 6.3 储物袋
- 材料/丹药大堆叠 + 软上限(初版不做容量折磨)。
- 法宝/功法实例占槽,默认上限 100(可灵石升级 → sink)。

## 7. 战斗引擎(即时自动结算)

每场战斗 = 一次确定性模拟(轻随机),**最多 30 回合**。

### 7.1 流程
1. 加载双方最终属性 + 功法栏(战技 ×3 + 心法被动)+ 法宝词条。
2. 按**身法**定先后手(高者先;每回合双方各行动一次)。
3. 每回合行动:从战技栏选可用技(法力够、不在 CD)→ 否则普攻。
4. 战斗结束:一方气血 ≤ 0,或满 30 回合 → 按剩余气血百分比判胜。
5. 输出**精简战斗日志**(关键 6~10 行)+ 胜负 + 奖励。

### 7.2 伤害公式
```
单次伤害 = max(1, 攻击 × 功法系数 × [1 − 防御 / (防御 + 300)]) × 暴击修正 × rand(0.9, 1.1)
暴击触发率 = 暴击 / (暴击 + 200)         (上限 50%)
暴击修正   = 1.5(触发) / 1.0(未触发)
每回合法力回复 = 最大法力 × 5%
普攻系数 = 1.0
```
`rand` 每场战斗用固定种子(可复现,便于调试与申诉)。

### 7.3 战技原型(初版示例)

| 战技 | 类型 | 效果 | 法力 | CD |
|---|---|---|---|---|
| 快剑斩 | 爆发 | 系数 1.8 单次 | 20 | 2 |
| 烈火诀 | 持续 | 每回合系数 0.6,持续 3 回合 | 30 | 4 |
| 定身符 | 控制 | 对方 1 回合无法行动 | 25 | 3 |
| 回春术 | 回血 | 回复最大气血 15% | 30 | 3 |
| 金钟罩 | 护盾 | 吸收下一次伤害 | 20 | 3 |

战技/法宝构成流派(爆发/持续/控制/续航),即时结算下仍有 build 深度。

## 8. 制造(炼丹 + 炼器)

- **炼丹**:草药 + 丹方 + 灵石(燃料) → 丹药。
- **炼器**:矿石/兽材 + 图纸 + 灵石 → 法宝(随机词条)。
- **定时完成**:`/craft` 下炉 → 写入 `crafting_jobs(start_at, finish_at)` → 惰性结算;`finish_at` 前可**灵石加速**(剩余时长 × 单价)。
- **熟练度**:炼丹/炼器各一条熟练度,升级提高**高品质产出概率**;品质 roll 受"根骨 + 熟练度"影响。
- 配方(丹方/图纸)来源:掉落、宗门商店、NPC 商店。

## 10. PvP(切磋 + 天梯)

- **发起**(群内):`@道友` 切磋 · 随机匹配(天梯分 ±段)· 挑战排行榜玩家。
- **结算**:取双方**当前配置快照**用 §7 引擎模拟,**异步出结果**(对方无需在线);纯竞技,**不掉任何资源**。
- **天梯**:ELO 式积分,初始 1000;`Δ = K × (结果 − 期望)`,K=32。段位映射:凡品 / 灵品 / 宝品 / 仙品 / 地仙(按分段)。v1 长期榜,**赛季制二期**。
- **限制**:每日切磋次数上限(初版 10 次/日,独立计数,不耗精力);**境界差过大限制匹配**,防虐菜。
- 奖励:积分 + 少量灵石/声望;失败无惩罚。

## 11. 宗门(v1 范围)

- 创建(需筑基 + 灵石)/ 申请加入 / 退出。
- 宗门等级、**贡献点**、职位(宗主 / 长老 / 弟子)。
- **全员福利**(随宗门等级):闭关速率 +、精力上限 +、掉落 +、离线上限 +。
- **宗门任务**(日常):产出贡献点。
- **宗门商店**:贡献点兑换功法残页 / 丹方 / 材料。
- **宗门战 / 擂台 → 二期。**

- emoji 点缀:⚔️ 战斗 · 🔮 法宝 · 💊 丹药 · 📖 功法 · 🪙 灵石 · 🧘 闭关 · 🐲 妖王。

### 12.3 群 / DM 分流
- **DM**:养成主场(面板、闭关、历练、背包、炼丹、突破)。
- **群**:切磋、世界Boss、排行、宗门播报。群内尽量用**单条可编辑消息 + 按钮**,严禁逐条刷屏。
- 群内点"养成"类操作 → 引导跳转 DM(deep link)。

### 12.4 文案语气(半文半白·正经仙侠)
示例:
- 出关:「道友闭关三个时辰,修为精进 1,250 点,丹田渐盈。」
- 突破失败:「天劫加身,道基微损,折损修为三成。来日再战。」
- 切磋胜:「一剑光寒,道友技高一筹,天梯积分 +18。」

## 13. 数据模型(SQLite)

> 静态数值(物品/妖兽/地图/功法/丹方/境界)进**配置文件**(`config/*.py` 或 JSON),不入库,便于平衡。库内只存玩家动态状态。

核心表(列为要点,实施时补全约束与索引):

```
users(tg_user_id PK, username, created_at, last_seen_at)

characters(
  user_id PK FK,
  root_bone INT,            -- 根骨
  spirit_root TEXT,         -- 灵根品类文案
  realm INT, realm_stage INT,
  cultivation INT,          -- 当前修为
  hp_base, mp_base, atk, def, spd, crit,   -- 基础六维(或由境界配置推导)
  stamina INT, stamina_updated_at INT,
  seclusion_start_at INT NULL, seclusion_place TEXT,
  spirit_stone INT,
  debuff_json TEXT,         -- 道基不稳等临时状态
  craft_alchemy_prof INT, craft_forge_prof INT
)

inventory(id PK, user_id FK, item_key TEXT, qty INT)            -- 堆叠类(材料/丹药/杂项)
item_instances(id PK, user_id FK, base_key TEXT, tier INT, affixes_json TEXT, equipped_slot TEXT NULL)  -- 法宝实例
skills_learned(user_id, skill_key, level)
skill_loadout(user_id, slot, skill_key)                          -- 心法×1 + 战技×3
recipes_known(user_id, recipe_key)
crafting_jobs(id PK, user_id FK, craft_type, recipe_key, start_at, finish_at, status)

sects(id PK, name UNIQUE, level, contribution_pool, leader_user_id, created_at)
sect_members(sect_id, user_id, role, contribution, joined_at)

pvp_ratings(user_id PK, rating INT, wins INT, losses INT, daily_count INT, daily_reset_at INT)

world_boss(id PK, chat_id, boss_key, total_hp, remaining_hp, spawn_at, expire_at, status)
world_boss_damage(boss_id, user_id, damage)

daily(user_id, last_checkin_at, streak)
battle_log(id PK, user_id, kind, summary_json, created_at)       -- 可选,精简留存
```

并发与一致性:
- aiosqlite + **WAL** 模式;对灵石/库存等写操作做**写串行化**(单写连接或每用户异步锁),避免竞态。
- 所有时间用**服务器时间戳**;惰性结算函数集中在 `services/time_settle.py`。

## 14. 技术架构

- **aiogram 3.x**:Router 分模块,F 过滤器,FSM(`StatesGroup`)处理多步流程;polling 起步。
- **aiosqlite**:异步访问 + WAL;迁移脚本管理 schema。
- **APScheduler**(AsyncIOScheduler):世界Boss 刷新/结算、每日重置、天梯日结。
- **惰性结算**:精力恢复、闭关修为、制造完成全部读时计算,不轮询。
- **配置驱动**:数值/掉落/物品/功法/妖兽集中配置,支持热调参。
- **防作弊**:服务器端权威计算、操作限频、回调幂等、不信任客户端输入。
- **可测**:战斗模拟、惰性结算、突破概率、掉落分布可写单元测试(确定性种子)。
- **部署**:单进程 + 单 SQLite 文件起步;后续可迁 webhook / Postgres。
- 建议目录:
  ```
  bot/        # 入口、Dispatcher、Router 注册
  handlers/   # 各指令与回调处理
  services/   # 战斗、结算、修炼、制造、PvP、宗门、Boss 业务逻辑
  models/     # DB 访问层
  config/     # 静态数值与物品/妖兽/功法/地图定义
  data/       # SQLite 文件
  tests/
  ```

## 16. 里程碑与实施拆分(供日后 `/batch` 并行)

模块与依赖(① 为关键路径前置):

1. **基础设施** — bot 启动、DB schema/迁移、配置加载、惰性结算工具、用户注册/测灵根。

4. **PvE** — 历练/秘境/妖王/掉落、世界Boss。 *(依赖 ③)*
5. **物品/制造/经济** — 储物袋、装备、炼丹炼器、NPC 商店。 *(依赖 ①②)*
6. **PvP** — 切磋、天梯、排行。 *(依赖 ③)*
7. **宗门** — 组织、贡献、福利、商店。 *(依赖 ①②)*
8. **交互层** — 指令、菜单、FSM、文案、群/DM 分流、防刷屏播报。 *(贯穿,依赖各业务)*
